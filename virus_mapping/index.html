Virus mapping guide

A guide for producing alignments from nanopore FASTQ data

You will:

  1. Identify a virus.
  2. Map against NCBI reference.
  3. Preform variant calling and annotation.
  4. Plot coverage, SNP frequencies and gene annotations.

Pre-requisites

  1. Mamba/Conda: See instruction for installing Mamba.

  3. Downloading a reference sequence from RefSeq.

Step 1 - Mapping with Minimap2

minimap2 -ax map-ont ~/refseq/virus/virus_reference.fasta  sequencing_reads.fastq | \
samtools view -S -b - |\
samtools sort - -o output_alignment.bam

Step 2 - Alignment coverage map

Run BEDtools to generate a coverage map: 

bedtools genomecov -ibam output_alignment.bam  -bga > genome_coverage.bed

Step 4 - Variant calling

Perform variant calling using freebayes. Add flags where appropriate:

FreeBayes Flags Explanation

-C or --min-alternate-count - Description: Sets the minimum number of observations that must support an alternate allele within a single individual to consider it in variant calling. This parameter helps to reduce false positives, especially important in noisy sequencing data. - Typical Usage: -C 10 implies that at least 10 reads must support an alternate allele before it is considered a valid variant. This threshold can be adjusted based on sequencing quality and expected coverage.

-q or --min-mapping-quality - Description: Specifies the minimum mapping quality of reads that are considered in the variant calling process. This flag helps filter out reads with poor alignment scores, which might indicate ambiguous alignments or sequencing artifacts. - Typical Usage: -q 20 means only reads with a mapping quality score of 20 or higher will be considered, helping ensure that only high-quality data contributes to the variant calling results.

-R or --min-alternate-fraction - Description: Sets the minimum fraction of observations that must support an alternate allele for it to be considered a variant. This setting helps avoid calling very rare variants that might be due to sequencing errors. - Typical Usage: -R 0.1 indicates that at least 10% of all reads at a position must support the alternate allele to count it as a variant.

--min-coverage - Description: Establishes the minimum number of reads covering a position required to evaluate it for variant calling. Ensuring sufficient coverage at a site is crucial for making reliable variant calls. - Typical Usage: --min-coverage 20 means that a site must be covered by at least 20 reads to be considered for variant calling, which helps reduce the likelihood of false positives due to low coverage.

freebayes -f [reference.fasta] output_alignment.bam > variants.vcf

# Call variants
bcftools mpileup -Ou -f [reference.fasta] sorted_mapped.bam | bcftools call -mv -Ob | bcftools view -Oz -o variants.vcf.gz

Step 5 - Generating a consensus sequence

# First, index the VCF file
bcftools index variants.vcf.gz

# Generate consensus
cat [reference.fasta] | bcftools consensus -f - variants.vcf.gz > consensus.fa

# Generate a bed file where coverage is below the threshold
awk -v min_depth=[min_depth] '$4 < min_depth {print $1"\t"$2"\t"$3}' genome_coverage.bed > low_coverage.bed

# Mask the consensus sequence
bedtools maskfasta -fi consensus.fa -bed low_coverage.bed -fo masked_consensus.fa

Step 5 - SNP annotation

# Download the appropriate database
snpEff download [genome_version]

# Annotate the VCF file
java -Xmx4g -jar snpEff.jar [genome_version] variants.vcf > annotated_variants.vcf
